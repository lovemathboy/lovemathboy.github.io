--- Day 22: Passwords ---

Part 1

Your boss has tasked you with the completely ethical task of hacking passwords.

Weirdly, they require each password to be a positive integer.

Luckily, from some social engineering, you have two pieces of information about every user:

- They have a favorite digit, and this digit appears as at least half the digits in the password

- They have a favorite number, and the password is divisible by their favorite number

In addition, passwords cannot contain any leading zeroes.

For each user, determine the smallest password they could have!

Your final answer is the sum of all passwords.

Example Input:

User 1:
Favorite Digit: 8
Favorite Number: 2

User 2:
Favorite Digit: 4
Favorite Number: 38

User 3:
Favorite Digit: 5
Favorite Number: 492

User 4:
Favorite Digit: 0
Favorite Number: 3

The smallest passwords to each user are 8, 494, 55596, and 30 respectively. The answer is 8 + 494 + 55596 + 30 = 56128

Note that 114 is not a valid password for User 2, as the digit "4" does not appear in at least half the digits.

Note that 03 is not a valid password for User 4, as leading 0s are not allowed. Also, 90, while a valid password for User 4, is not the smallest password.

What is the sum of the shortest passwords in your input?

"PASSWORDS ARE AT LEAST 8 DIGITS LONG!"

Oh right, there's another restriction on passwords - All passwords are between 8 and 16 digits (inclusive)

Count how many valid passwords (between 8 and 16 digits, inclusive) each user has, and sum them together!

Example Input:

User 1:
Favorite Digit: 8
Favorite Number: 2

User 2:
Favorite Digit: 4
Favorite Number: 38

User 3:
Favorite Digit: 5
Favorite Number: 492

User 4:
Favorite Digit: 0
Favorite Number: 3

If you were to only count passwords between 1 and 5 digits long: There are 1035, 59, 1, and 201 possible passwords for each user. The total number of passwords is 1296

If you were to count the passwords between 8 and 16 digits long: There are 457351181280, 24071117620, 633963632, and 109262694015 possible passwords for each user. The total number of passwords is 591318956547

Count how many valid passwords (between 8 and 16 digits, inclusive) each user has, and sum them together!
