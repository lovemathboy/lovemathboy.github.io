--- Day 17: Networking ---

Part 1

Somehow, your boss decided to entrust you with the task of managing a network of terminals - Your boss has too much faith in those remedial lessons.

The network of terminals can be described as a directed graph, where an edge (x -> y) means terminal x can send a message to terminal y

Each terminal is given a unique integer as an id. Due to security measures, a higher terminal id can never send a message to a lower terminal id

Your boss has tasked you with determining the longest chain of edges in this terminal system.

Example Input:

0 -> 3
1 -> 2
2 -> 3
3 -> 5
1 -> 3
2 -> 6
4 -> 7

The longest communication chain has 4 terminals.
One possible path is 1 -> 2 -> 3 -> 5

What is the length of the longest path in the following network?

You need to add 3 additional directed edges.
One possible solution is adding the following 3 edges: 5 -> 6, 6 -> 4, and 7 -> 1.

Assume that every terminal id not mentioned in the input is inactive and cannot be connected to (and hence should be ignored)

What's the minimum number of directed edges you need to add to the graph to allow every vertex to (eventually) visit another vertex?
